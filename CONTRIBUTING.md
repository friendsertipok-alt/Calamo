# Contributing to Calamo

Thank you for your interest in contributing to Calamo! We welcome contributions from developers, researchers, and writers to help make automated academic document generation better and more compliant with international standards.

## How to Contribute

### 1. Report Bugs
If you find a bug, please create an Issue on GitHub with:
- A clear, descriptive title.
- Steps to reproduce the bug.
- Expected vs actual behavior.
- Screenshots or error logs (if applicable).

### 2. Suggest Features
If you have ideas for new features, open an Issue to discuss it first. Let us know:
- What problem this feature solves.
- How you imagine the implementation.

### 3. Submit Pull Requests
To submit code changes:
1. Fork the repository.
2. Create a new branch from `main` (e.g., `feature/awesome-new-capability` or `bugfix/fix-docx-align`).
3. Make your changes. Ensure you follow our code style and rules in `AGENTS.md`.
4. Ensure all tests pass.
5. Submit a Pull Request targeting the `main` branch.

## Code Style & Standards

- **Python**: Follow PEP 8 guidelines. Write type annotations for function signatures.
- **HTML/CSS/JS**: Keep scripts clean, modular, and performant. Use semantic tags.
- **GOST Standards**: Ensure any changes to document structure or formatting comply with GOST guidelines.
