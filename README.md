# Calamo — ИИ-генератор академических работ по ГОСТу

<p align="center">
  <img src="frontend/assets/favicon.webp" alt="Calamo Logo" width="120">
</p>

Calamo — это современная open-source веб-платформа для автоматической генерации академических работ (курсовые работы, рефераты, дипломные работы, отчеты по практике) в строгом соответствии со стандартами ГОСТ (включая ГОСТ 7.0.100-2018 для библиографии).

Проект использует продвинутый LLM-пайплайн (Google Gemini / ProxyAPI), автоматический сбор источников в Google Scholar и интеллектуальную генерацию аналитических графиков и таблиц, бесшовно интегрируемых в формат Microsoft Word (`.docx`).

---

## ✨ Ключевые возможности

- **Полное соответствие ГОСТ**: Форматирование шрифтов, абзацев, отступов, заголовков, таблиц, рисунков и списков литературы по ГОСТ 7.0.100-2018.
- **Интеллектуальный сбор источников**: Интеграция с научными базами данных и Google Scholar для поиска реальных, актуальных источников.
- **Генерация визуализаций**: Создание графиков (Matplotlib) и таблиц на основе сгенерированных данных перед написанием основного текста с последующим анализом в теле работы.
- **Отсутствие Markdown**: Специально обученный пайплайн гарантирует отсутствие разметки `#`, `**` и т.д. в результирующем документе.
- **Интеграция с RollyPay**: Удобная и безопасная платежная система PCI DSS.
- **Telegram Уведомления**: Логирование ключевых этапов генерации и бэкапов в Telegram-канал/чат.

## 🛠 Технологический стек

### Бэкенд
- **Язык**: Python 3.10+
- **Фреймворк**: FastAPI
- **СУБД**: SQLite / PostgreSQL (через SQLAlchemy и Alembic)
- **Фоновые задачи**: ARQ (Redis)
- **Генерация документов**: python-docx
- **Инференс**: Google Generative AI (Gemini) / OpenAI (через ProxyAPI)

### Фронтенд
- **Структура**: Vanilla HTML5, семантическая верстка, SEO-оптимизация
- **Стили**: Vanilla CSS (премиальный темный интерфейс с микроанимациями)
- **Логика**: Vanilla JavaScript (ES6+)

## 🚀 Быстрый старт (Локально)

### Требования
- Python 3.10+
- Node.js (для сборки фронтенда, если применимо, либо Nginx)
- Redis (для работы фоновых задач)

### 1. Клонирование репозитория
```bash
git clone https://github.com/friendsertipok-alt/Academicpro.git
cd Academicpro
```

### 2. Настройка переменных окружения
Создайте файл `.env` в корневой директории бэкенда на основе `.env.example`:
```bash
cp backend/.env.example backend/.env
# Откройте backend/.env и заполните ваши API ключи
```

### 3. Запуск бэкенда
```bash
cd backend
python -m venv venv
source venv/bin/activate  # На Windows: venv\Scripts\activate
pip install -r requirements.txt
uvicorn app.main:app --reload
```

### 4. Запуск фронтенда
Фронтенд представляет собой статические файлы. Для локального запуска достаточно открыть `frontend/index.html` в браузере или использовать любой локальный веб-сервер:
```bash
cd ../frontend
python3 -m http.server 8080
```
Перейдите на [http://localhost:8080](http://localhost:8080).

## 🏗 Архитектура пайплайна генерации

Пайплайн генерации разделен на строго последовательные этапы для обеспечения высокого академического качества:

1. **Анализ темы**: LLM разбирает тему, формирует структуру разделов и находит релевантные научные статьи.
2. **Генерация черновика**: Создается подробный план и список литературы по ГОСТ.
3. **Генерация визуальных объектов**: Сначала создаются спецификации графиков (`ChartSpec`) и таблиц (`TableSpec`), чтобы текст в главах ссылался на конкретные данные.
4. **Написание глав**: Генерация контента глав с интеграцией аналитических абзацев и отсылок к созданным визуализациям.
5. **Сборка DOCX**: Компиляция всех материалов в итоговый `.docx` файл с ГОСТ-стилизацией.
